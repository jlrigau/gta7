# Content suitability checklist (blocking) — assets

Before **proposing** an asset, you must have **looked at it** and checked every point.
When in doubt on any of them: **don't propose**.

The bar is set by the game's audience — read `GAME.meta.audience` (e.g.
`{ minAge: 9, notes: "kids, gentle, no violence" }`). The younger / more sensitive
the audience, the stricter the filtering.

## ✅ Must be TRUE
- [ ] The image was **actually opened and looked at** (Read on the downloaded preview).
- [ ] **On-theme** and consistent with the game (`GAME.meta`, `style.css`).
- [ ] **Art style** matches (e.g. pixel-art LPC), not clashing.
- [ ] **License** permissive and attributable (CC0 / CC-BY / CC-BY-SA / OGA-BY / GPL),
      with **author + URL** noted for attribution.

## ⛔ Reject immediately when the audience is general / children (presence = reject)
- Violence, injuries, **blood / gore**.
- **Weapons**, scary monsters, **horror**, distressing mood.
- **Sexual or suggestive** content, nudity.
- **Drugs / alcohol / tobacco**, gambling.
- **Slurs**, hateful symbols, adult themes.
- Off-theme (nothing to do with the game).

> Adjust this list to the actual game. For a mature-rated game, relax what's
> appropriate — but never relax the **license** requirement, and always **look at the
> image** before proposing it.

## Golden rule
> **When in doubt, don't propose it.** One safe option beats three doubtful ones.
